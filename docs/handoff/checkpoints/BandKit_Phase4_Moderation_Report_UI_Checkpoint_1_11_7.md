## Scope of this slice

```text
src/modules/RealChat.ts                 (report action → reason picker → POST /reports)
src/modules/RealModerationReports.ts    (new — staff reports queue on /moderation)
src/main.ts                             (init)
src/routes/pages.ts                     (mount on /moderation)
src/locales/en|ru/chats.json            (chats.report.*)
src/locales/en|ru/moderation.json       (moderation.reports.*)
src/locales/bundles.ts                  (regenerated by build)
```

### Reporter side (chat)

- The existing message context menu's **Report** item now opens an in-place reason picker (spam / harassment / threats / scam / suspicious link / impersonation / other — stable keys, localized labels) reusing the same floating menu, so no new UI/CSS.
- Choosing a reason posts to `POST /reports` with `object_type: chat_message` and `object_id`. On success the menu shows the spec's confirmation copy — "Report sent. We saved the message and its context for review." — then closes. No promise of a specific outcome.

### Staff side (/moderation)

- `RealModerationReports.ts` mounts on the `/moderation` page and fetches `GET /reports`. Non-staff get 403 → the panel empties itself, leaving the existing mock queue untouched.
- Renders the real report queue (object type, reason, reporter) with a localized state `<select>` per case (created / triage / in_review / action_required / resolved / rejected / closed) that PATCHes `/reports/:id`. Shared classes only.
