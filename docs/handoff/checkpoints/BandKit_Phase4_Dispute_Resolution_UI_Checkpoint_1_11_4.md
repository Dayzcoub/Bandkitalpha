# BandKit — Phase 4 Dispute Resolution UI Checkpoint 1.11.4

## Status

Accepted checkpoint.

Reputation-domain slice: the organizer-facing UI for the dispute flow shipped in 1.11.3. The engagements roster now shows each reliability record's dispute state and lets the organizer resolve an open dispute.

## Scope of this slice

```text
src/modules/RealEventEngagements.ts
src/locales/en/events.json
src/locales/ru/events.json
src/locales/bundles.ts               (regenerated by build)
```

- Each reliability record in the roster panel shows its dispute state as a localized marker (`dispute: open / upheld / retracted`); a record with no dispute keeps the plain `disputed` marker if flagged.
- When a record's dispute is `open`, the panel offers **Uphold** and **Retract** buttons wired to `PATCH /reliability-events/:id/dispute`. On success it reloads the engagement's records and shows a status message.
- Authz-honest: only resolution is offered here, because resolving is the organizer/staff side. Opening a dispute is the subject's action and belongs on a subject-facing surface (a later slice); the roster viewer is the event's manager, who would be forbidden from opening.
- Localized via `events.reliability.disputeState.*` and `events.reliability.resolve*` (en/ru). Built from existing shared classes — no new CSS.


Backend dispute endpoints themselves were verified in 1.11.3.
